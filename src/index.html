32. STATIC

Conteúdo estudado do canal: Bro Code

  • → É uma keyword que define propriedades ou métodos que pertence a uma class ao invés de um objeto criado por aquela classe;
  • → Ou seja: ao inés de criar const MathUtil = new MathUtil(); para acessar um possível objeto, eu simplesmente posso criar uma static keyword dentro da class da seguinte forma: static PI = 3.14159 e acessá-la fora da classe da seguinte forma: console.log(MathUtil.PI);
  • → De resumo: tudo estático pertence a classe em si!
  • → Se tentarmos acessar a static, não conseguiremos, já que a classe não possui o static como uma propriedade. Temos que usar o nome da CLASSE para acessá-la;

34. SUPER

Conteúdo estudado do canal: Bro Code

  • → É uma keyword usada para chamar o construtor ou acessar as propriedades e métodos de um parent. É semelhante ao this, sendo este para o this object e o super para the parent;
  • → O que isso significa é que, de acordo com o vídeo, antes de chamar as this keywords, caso a class-pai tenha um constructor, é necessário chamar a super() keyword;
  • → A ideia aqui é não se repetir. Caso 5 sub-classes tenham, por exemplo, nome e idade, para não nos repetirmos em cada uma delas, apenas colocamos as propriedades no constructor(nome, idade) {...} e na hora de colocar o super dentro de cada sub-classe, dentro de cada construtor, usamos super(nome, idade);

44. Error Handling

Conteúdo estudado do canal: Bro Code

  • → É um objeto (sim, um objeto) que representa um erro que ocorreu. Normalmente ocorre com input ou na parte de estabelecer um conexão. Mas o porque de precisar desse tratamento? Normalmente, quando um erro ocorre, todo o programa para e espera a correção do mesmo. Há erros que precisam ser corrigidos na hora, porém outros são passíveis de espera;

Para testá-los, usamos 3 tipos de "blocos":

  • → try{}: todo código colocado dentro dos {} é do tipo que poderá causar algum erro futuro;
  • → catch(){}: (o parâmetro dentro é para pegar um error oject, ou seja, error) { aqui podemos colocar console.log(error) para descobrir qual erro é }
  • → finally{}: todo código colocado dentro dos {} é executado;

45. DOM

Conteúdo estudado do canal: Bro Code

  • → Aqui entra alguns conceitos sobre DOM (Document Object Model);
  • → Sobre os seletores de elementos, dizemos que são métodos usados para encontrar e manipular elementos HTML, podendo ser um ou vários;

Para testá-los, usamos 5 tipos:

  • → document.getElementById(): retorna um ELEMENT ou NULL;
  • → document.getElementsClassName(): retorna uma HTML COLLECTION;
  • → document.getElementsByTagName(): retorna uma HTML COLLECTION;
  • → document.querySelector(): retorna o FIRST ELEMENT ou NULL;
  • → document.querySelectorAll(): retorna uma NODELIST;
  • → Passando um pouco mais para frente, temos as classList, que é uma propriedade usada para interagir com uma lista de classes de um elemento (normalmente CSS);

Temos algumas opções (usaremos um button para teste):

  • → .add(): adiciona classe(s) a um determinado elemento;
  • → .remove(): remove classe(s) de um determinado elemento;
  • → .toggle(): funciona como se fosse um interruptor, "adicionando e removendo" classe(s);
  • → .contains('className'): verifica se a className está contida dentro daquele elemento;
  • → .replace('oldClass', 'newClass'): troca a(s) classe(s) de um determinado elemento;

47. Promisses

Conteúdo estudado do canal: Bro Code

  • → É um objeto que administra asynchronous operations. Operações que podem tomar um certo tempo, e que não queremos que elas pausem todo o resto dos outros códigos;

  • → Para escrever: new Promisse((resolve, reject) => {asynchronous code});
