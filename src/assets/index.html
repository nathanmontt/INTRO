Internet

How does the Internet work?

Conteúdo resumido do vídeo: How the Internet Works in 5 Minutes, por Aaron.

  • Podemos pensar na Internet como um cabo, um fio, um wire;
  • Sendo um cabo, ele está (pelo menos no caso da Internet) debaixo da terra sendo um fio de cobre, por exemplo. Mas também temos os satélites, por exemplo;
  • Além disso, temos computadores especiaias, os servidores. São eles que estão diretamente conectados aos wires, além de, dentro deles (os servers), termos hardrives que armazenam as web pages;
  • Só que ter apenas isso não faz a comunicação entre tudo acontecer de forma mágia. Precisamos de algo. Com isso, os servidores possuem algo chamado IP (ou protocolo de Internet). Se ficar complicado de imaginar, é só pensar em um endereço;
  • O problema é que, se pesquisarmos, por exemplo, endereços de IP, vamos nos deparar com 72.14.205.100. E o que é esse valor??? Bom, já que não conseguimos entender esse valor, é dado um nome a ele. Esse nós chamamos de google.com;
  • E para se comunicar entre si, no caso eu e minha tia, por exemplo? Bom, nossos PC's não são conectados diretamente nos wires, então precisamos de um meio termo para isso. Surge então as ISPs, Internet Service Providers, ou só Provedores de Internet (VIVO, TIM, Claro, e por aí vai...). Mandamos mensagens "por elas", elas mandam "pela Internet" e chega até o destino;
  • Lembrando que ela a mensagem não sai inteira do seu PC até o destinatário. Ela é quebrada em packets, ou pacotes. No fim, ao ir chegando ao outro cliente (chamamos dessa forma chique mesmo), ela vai se reconstruindo;
  • Mas e se eu quiser mandar um email a uma pessoa e, na minha casa, meu pai quiser fazer o mesmo? Meu email não vai colidir com o dele??
  • Na verdade, não. O motivo disso são os endereços IP e os roteadores. Eles são responsáveis para que isso não ocorra. Para os servidores, eles tem a finalidade de guiar os pacotes ao seu devido destino. E como eles fazem isso? Através dos endereços IP (são únicos, ok?);

What is HTTP?

Conteúdo resumido do vídeo: What is HTTP and Why Web Is Built On It?, por Hooman Mardox.

  • De acordo com o vídeo, HTTP significa HyperText Transfer Protocol. O HyperText é possível entender como os links que estão presentes dentro da página;
  • Então, de forma resumida, HTTP é um protocolo que transfere esses HyperTexts do servidor web para os browsers;

What is Domain Name?

Conteúdo resumido do vídeo: What Is a Domain Name | Domain Names Explained, por Hostinger Academy.

  • Se procurarmos por endereços IP na Internet, veremos uma baita quantidade de números, como 8.8.8.8, 8.8.8.4, e assim por diante. Mas esses número tem um significado. Para o PC. Para nós, não de cara;
  • Eles são os nomes dos sites. Mas não vamos ficar sabendo de todos os IPs de cabeça, e é para isso que o DNS (Domain Name Server) serve. QUando digitamos na barra de pesquisa de algum browser, é enviado uma request para o DNS que sai em busca desse nome. Quando o acha, compara com os endereços já registrados e, se compatíveis, retorna a pesquisa (no caso, vai em direção ao site escolhido);

HTML

1. Introduction to HTML

Conteúdo estudado do canal: Bro Code

  • HTML significa Hyper-text Markup Language. A ideia é que tags (essas que estão fazendo o texto ficar em negrito ou itálico), também conhecidas como - pasmem - markups, são usadas em web browsers para dar display em certos elementos, como por exemplo - pasmem mais uma vez - todo esse conteúdo;
  • Se ainda não deu para entender, imagine que você queira construir uma casa. O HTML seria a fundação dela. E aí entramos com a CSS (Cascading Style Sheet), que seria o equivalente às decorações, design de interiores e cores da casa.
